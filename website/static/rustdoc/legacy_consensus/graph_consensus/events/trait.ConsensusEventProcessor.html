fn reset(&mut self, pool_size: usize)

Reset (or set) the parameters of the consensus engine. All events not yet returned by next_committed_payload will be re-inserted and processed with the new consensus parameters.

Reset points and their parameters must be strictly agreed upon between participants. Therefore, in practice, this method will likely be triggered only at the beginning or when a special message was just returned by next_committed_payload.

fn process_verified_event(&mut self, event: VerifiedEvent<Self::Payload>)

Process a new event. We assume that all the necessary verifications have been done already.

fn get_processed_event(
    &self,
    event_hash: &HashValue
) -> Option<&VerifiedEvent<Self::Payload>>

Look up whether an event was already processed. This is used for event chaining verifications before insertion.

fn next_committed_payload(&mut self) -> Option<Committed<Self::Payload>>

Iterate and return on the next committed payload to process. This will trigger clean-ups.
